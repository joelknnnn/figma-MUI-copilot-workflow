Overview

This guide will show you how to convert Figma designs into production-ready React components using GitHub Copilot Pro with Figma MCP integration and Material UI.

What you'll learn:
  • How to set up GitHub Copilot Pro with Figma MCP
  • How to use Copilot Chat to generate components from Figma designs
  • Best practices for design-to-code conversion
  • Common pitfalls and how to avoid them

Key Tools

  • GitHub Copilot Pro: AI pair programmer in VS Code
  • Figma MCP: Model Context Protocol for Figma integration
  • Material UI v6: React component library
  • Vite: Fast build tool for React
  • TypeScript: Type-safe JavaScript

Prerequisites

Required Subscriptions

  • GitHub Copilot Pro subscription ($10/month)
  • Figma account (free or paid)

What You Need

  • macOS, Windows, or Linux
  • Basic command line knowledge (copy/paste commands)
  • VS Code installed
  • GitHub account
Important: This guide assumes you're starting from scratch. If you already have some tools installed, you can skip those steps.

Part 1: System Setup (One-time)

Do these steps once to set up your development environment.

1. Install Homebrew (macOS/Linux only)

Open Terminal and paste:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Windows users: Skip Homebrew. Use the official installers from each tool's website.

2. Install Node.js and npm

macOS/Linux:

brew install node

Windows: Download from nodejs.org

Verify installation:

node --version
npm --version

3. Install VS Code

  • Download from code.visualstudio.com
  • Install and open VS Code

4. Install GitHub Copilot in VS Code

  1. Open VS Code
  2. Click Extensions icon (left sidebar) or press Cmd+Shift+X (Mac) / Ctrl+Shift+X (Windows)
  3. Search for "GitHub Copilot"
  4. Click Install on both:
    • GitHub Copilot
    • GitHub Copilot Chat
  5. Sign in with your GitHub account when prompted
  6. Verify Copilot Pro subscription is active
Success indicator: You should see a GitHub Copilot icon in VS Code's left sidebar

5. Install GitHub CLI

macOS:

brew install gh

Windows:

winget install --id GitHub.cli

Linux: Follow instructions at github.com/cli/cli

Login to GitHub:

gh auth login
  • Choose: GitHub.com
  • Choose: HTTPS
  • Choose: Login with a web browser
  • Follow browser prompts

6. Configure Git

git config --global user.name "Your Name"
git config --global user.email "[email protected]"

7. Install Figma Desktop

  • Download from figma.com/downloads
  • Install and sign in
  • Important: Keep Figma Desktop running (MCP requires it)

8. Enable Figma Dev Mode MCP

  1. Open Figma Desktop (not web version)
  2. Click Figma menu â†’ Preferences
  3. Go to Experimental Features or Advanced tab
  4. Enable "Dev Mode MCP Server"
  5. Note the MCP URL: http://127.0.0.1:3845/mcp
Critical: Figma Desktop must remain open while using MCP. The web version won't work.

9. Enable Copilot Instruction Files (Optional but Recommended)

  1. Open VS Code
  2. Press Cmd+Shift+P (Mac) / Ctrl+Shift+P (Windows)
  3. Type: Preferences: Open User Settings (JSON)
  4. Add this line to your settings:
{
    "github.copilot.chat.codeGeneration.useInstructionFiles": true
}
Note: This allows Copilot to read project-specific instruction files. We'll configure MCP in the next section when we create the project.

7. Open Project in VS Code

8. Create Copilot Instructions File

Create a file called .github/copilot-instructions.md in your project:

mkdir -p .github
nano .github/copilot-instructions.md

Paste this content:

# GitHub Copilot Instructions for Figma to Material UI Conversion

## Project Context
This project converts Figma designs to React components using Material UI v6 and TypeScript.

## Code Generation Rules

### 1. Always Use Material UI Components
- Use MUI components instead of native HTML elements
- Import from: `@mui/material` and `@mui/icons-material`
- Example: `Button`, `TextField`, `Box`, `Container`, `Typography`

### 2. TypeScript Types
- All components must have proper TypeScript types
- Define prop interfaces with `interface ComponentNameProps`
- Use React.FC or explicit function typing

### 3. Theme Integration
- Use theme spacing: `theme.spacing(2)` not hardcoded pixels
- Use theme colors: `theme.palette.primary.main`
- Access theme with: `import { useTheme } from '@mui/material/styles'`

### 4. Responsive Design
- Use MUI's sx prop for styling
- Use breakpoints: `{ xs: 12, sm: 6, md: 4 }` for Grid
- Mobile-first approach

### 5. File Organization
- Components go in: `src/components/[ComponentName].tsx`
- One component per file
- Export as default

### 6. Figma Design Extraction
When fetching from Figma MCP:
1. Extract all design tokens (colors, spacing, typography)
2. Map Figma components to closest MUI equivalents
3. Preserve exact spacing and visual hierarchy
4. Extract text content and alt text for images

### 7. Code Quality
- Use functional components with hooks
- Prefer composition over inheritance
- Keep components focused and single-purpose
- Add comments for complex logic

### 8. Styling Approach
- Use `sx` prop for component-level styles
- Create theme customizations in `src/theme.ts`
- Avoid inline styles
- Use MUI's emotion-based styling

## Example Component Structure

```typescript
import React from 'react';
import { Box, Typography, Button } from '@mui/material';
import { useTheme } from '@mui/material/styles';

interface MyComponentProps {
  title: string;
  onAction?: () => void;
}

const MyComponent: React.FC = ({ title, onAction }) => {
  const theme = useTheme();

  return (
    <Box sx={{ p: theme.spacing(3) }}>
      <Typography variant="h4" gutterBottom>
        {title}
      </Typography>
      <Button 
        variant="contained" 
        onClick={onAction}
        sx={{ mt: theme.spacing(2) }}
      >
        Click Me
      </Button>
    </Box>
  );
};

export default MyComponent;
```

## When Converting from Figma
1. Ask Copilot to fetch the Figma design via MCP
2. Request extraction of design tokens
3. Generate component with exact visual match
4. Review and refine spacing/colors as needed

Save and exit (Control+O, Enter, Control+X)

9. Configure Figma MCP Connection

CRITICAL STEP: Create MCP configuration so Copilot can connect to Figma.

Create the .vscode directory and mcp.json file:

mkdir -p .vscode
nano .vscode/mcp.json

Paste this exact content:

{
    "servers": {
        "figma-mcp": {
            "url": "http://127.0.0.1:3845/mcp",
            "type": "http"
        }
    },
    "inputs": []
}

Save and exit (Control+O, Enter, Control+X)

Why this is needed: GitHub Copilot requires a project-specific .vscode/mcp.json file to connect to MCP servers. Without this file, Copilot cannot access Figma's MCP server, even if Figma Desktop is running.
Important notes:
  • The server name "figma-mcp" can be anything you want
  • The URL must match Figma's MCP server: http://127.0.0.1:3845/mcp
  • The type must be "http" for HTTP-based MCP servers
  • Figma Desktop must be running for this to work

10. Commit Configuration Files

git add .github/copilot-instructions.md .vscode/mcp.json
git commit -m "Add Copilot instructions and MCP configuration"
git push

11. Restart VS Code

Important: After creating .vscode/mcp.json, you must restart VS Code for Copilot to recognize the MCP server.

  • Close VS Code completely
  • Reopen with code . or through the application
  • Wait for Copilot to initialize (check status bar)

12. Test the Dev Server

npm run dev

Open browser to: http://localhost:5173

You should see the Vite + React default page.

Tip: Keep this terminal running. Open a new tab (Cmd+T) for other commands.

4. Use Copilot Chat to Generate Component

Open Copilot Chat:

  • Click the GitHub Copilot icon in VS Code sidebar
  • Or press Cmd+Shift+I (Mac) / Ctrl+Shift+I (Windows)

Example prompt:

I have a Figma design at [paste your Figma URL].

Please:
1. Connect to the Figma MCP server and fetch this design
2. Extract all design tokens (colors, spacing, typography, dimensions)
3. Create a React component using Material UI v6 components
4. Match the visual design exactly including all spacing and colors
5. Use TypeScript with proper type definitions
6. Save the component as src/components/DashboardCard.tsx
7. Follow all rules in .github/copilot-instructions.md

Make sure to:
- Use theme.spacing() for all spacing values
- Use MUI's sx prop for styling
- Map design elements to appropriate MUI components
- Add TypeScript interfaces for all props
Important: Copilot needs explicit instructions to use MCP. Always mention "Connect to Figma MCP" or "use Figma MCP server" in your prompts.

5. Review Generated Code

  • Copilot will show the generated component code
  • Review the code in the chat window
  • Click "Insert at Cursor" or "Apply in Editor"
  • Or copy the code manually to a new file

6. Test in Browser

  • Make sure dev server is running (npm run dev)
  • Import your new component in src/App.tsx
  • Check http://localhost:5173
  • Verify the component matches your Figma design

7. Iterate and Refine

If something doesn't look right, ask Copilot to adjust:

The spacing between the title and description is too tight. 
Use theme.spacing(3) instead of theme.spacing(1).

The button should be variant="contained" not "outlined".

Common Pitfalls & Solutions

Copilot Can't Connect to Figma MCP

Most Common Cause: Missing .vscode/mcp.json file!

Step-by-step fix:

  1. Verify the file exists: 
    ls -la .vscode/mcp.json
    If you get "No such file or directory", you need to create it (see Part 2, Step 9)
  2. Verify file contents: 
    cat .vscode/mcp.json
    Should show: 
    {
    "servers": {
        "figma-mcp": {
            "url": "http://127.0.0.1:3845/mcp",
            "type": "http"
        }
    },
    "inputs": []
}
  3. Restart VS Code after creating or modifying the file
  4. Verify Figma Desktop is running (not just web browser)
  5. Check MCP is enabled in Figma Preferences â†’ Experimental Features

Other solutions if still not working:

  • Restart Figma Desktop
  • Check if port 3845 is blocked by firewall
  • Try accessing http://127.0.0.1:3845/mcp in your browser to test if Figma's server is running
  • Check VS Code Output panel (View â†’ Output â†’ GitHub Copilot) for error messages

Copilot Not Following Instructions

Solutions:
  • Verify .github/copilot-instructions.md exists
  • Make sure VS Code setting github.copilot.chat.codeGeneration.useInstructionFiles is true
  • Be more explicit in your prompts: "Follow the rules in copilot-instructions.md"
  • Reference specific sections: "Use the TypeScript types pattern from our instructions"

Generated Code Uses Wrong MUI Version

Solution: Explicitly mention version in prompts 
Use Material UI v6 (latest) components and styling patterns

Solution: Install Node.js first

Can't Push to GitHub

Solutions:
  • Make sure you're logged in: gh auth status
  • First push needs: git push -u origin main

Solution: Start the dev server

Changes Not Showing in Browser

Solutions:
  • Check browser console for errors (F12)
  • Make sure dev server is running
  • Hard refresh: Cmd+Shift+R (Mac) / Ctrl+Shift+R (Windows)
  • Clear browser cache
  • Check if component is properly imported in App.tsx

TypeScript Errors in Generated Code

Solution: Ask Copilot to fix 
There are TypeScript errors in this component. Please fix:
- Add proper type definitions for all props
- Ensure MUI component props are correctly typed
- Fix any 'any' types

MUI Components Not Styled Correctly

Solution: Verify theme setup
  • Check if @emotion/react and @emotion/styled are installed
  • Ensure components use sx prop, not inline styles
  • Ask Copilot: "Refactor this to use MUI's sx prop and theme values"

Tips & Best Practices

Using Copilot Chat Effectively

  • Be specific: Include exact Figma URLs and component names
  • Reference files: Use @filename to give Copilot context (e.g., @App.tsx)
  • Iterate: Start with basic structure, then refine in follow-up prompts
  • Ask for explanations: "Explain why you chose this MUI component"

Managing Copilot Context

  • Select code: Highlight code in editor before asking questions
  • Use workspace context: Copilot sees your open files
  • Reference instruction file: Mention "Follow copilot-instructions.md" in prompts

Multiple Terminal Tabs Strategy

  • Tab 1: Dev server (npm run dev) - leave running
  • Tab 2: Git commands and other CLI tasks
  • Tip: Press Cmd+T (Mac) / Ctrl+T (Windows) for new tab

Use VS Code's Built-in Git Features

Instead of terminal commands, you can use VS Code's GUI:

  1. Click Source Control icon (left sidebar)
  2. Stage files with + icon
  3. Type commit message in text box
  4. Click âœ“ checkmark to commit
  5. Click ... menu â†’ Push

Iterative Refinement Prompts

Examples of good follow-up prompts:

// Spacing adjustment
"Increase spacing between header and content section to 32px"

// Component refinement
"Make the card elevation more subtle - use elevation={2} instead of 4"

// Responsive adjustment
"Make this component stack vertically on mobile (xs/sm breakpoints)"

// Type safety
"Add proper TypeScript types for the onSubmit callback"

// Accessibility
"Add proper ARIA labels to all interactive elements"

Keep Figma Organized

  • Name your frames clearly (matches component names)
  • Use Figma variables for colors (easier token extraction)
  • Document spacing using Figma's spacing annotations
  • Group related elements in frames
  • Use consistent naming conventions

Component Library Pattern

Create a component library structure:

src/
  components/
    common/          # Reusable components
      Button.tsx
      Card.tsx
    features/        # Feature-specific components
      Dashboard/
        DashboardCard.tsx
        DashboardHeader.tsx
    layout/          # Layout components
      Header.tsx
      Sidebar.tsx

Theme Customization

Extract Figma design tokens to a theme file:

// src/theme.ts
import { createTheme } from '@mui/material/styles';

export const theme = createTheme({
  palette: {
    primary: {
      main: '#1976d2', // from Figma
    },
    secondary: {
      main: '#dc004e',
    },
  },
  spacing: 8, // base spacing unit
  typography: {
    fontFamily: '"Roboto", "Helvetica", "Arial", sans-serif',
    h1: {
      fontSize: '2.5rem',
      fontWeight: 500,
    },
  },
});
